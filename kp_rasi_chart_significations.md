## 1. கே.பி. ஜோதிட விதிகளின் கட்டமைப்பு (KP Astrology Rules Structure)

புத்தகத்தில் உள்ள விளக்கங்களின்படி, ஒரு கிரகம் அல்லது பாவம் (House Cusp) கொடுக்கும் பலன்கள் பின்வரும் மூன்று காரணிகளால் தீர்மானிக்கப்படுகின்றன:
1. **நின்ற பாவம் (Occupied House - $X$):** கிரகம் அல்லது பாவ முனை அமைந்துள்ள வீடு.
2. **சார நாதன் நின்ற பாவம் (Star Lord's Occupied House - $Y$):** அந்த கிரகத்தின் அல்லது பாவ முனையின் நட்சத்திர அதிபதி (Star Lord) அமர்ந்துள்ள வீடு.
3. **12-ம் பாவ விலக்கு விதி (12th House Negation Rule / Bhava Subra):** ஒரு வீடு செயல்படும் போது, அதற்கு முந்தைய வீடு (12-வது வீடு) அந்த பாவத்தின் பலனைத் துண்டிக்கும் அல்லது கெடுக்கும்.

### பாவ விலக்கு விதி (Negation Formula):
- கிரகம் நின்ற வீடு $X$, அதன் சார நாதன் நின்ற வீடு $Y$ என்று வைத்துக் கொள்வோம்.
- $X$ என்பது $Y$-க்கு 12-வது வீடாக இருந்தால் (அதாவது $X = Y - 1$), $Y$-ன் பலன்கள் துண்டிக்கப்பட்டு **$X$ மட்டுமே செயல்படும்**.
- $Y$ என்பது $X$-க்கு 12-வது வீடாக இருந்தால் (அதாவது $Y = X - 1$), $X$-ன் பலன்கள் துண்டிக்கப்பட்டு **$Y$ மட்டுமே செயல்படும்**.
- மற்ற நிலைகளில் இரண்டு வீடுகளும் இணைந்து செயல்படும் ($Y, X$).

## 3. ஆண்ட்ராய்டு ஆப்பில் டைனமிக் முறையில் கணக்கிடுதல் (Dynamic Client Implementation)

உங்களது ஆண்ட்ராய்டு குறியீட்டில் [KpChartDialog.kt](file:///Users/wohozo/Documents/Astro Eleven/astroeleven/android/app/src/main/java/com/astroeleven/user/ui/chat/KpChartDialog.kt)-ல் இந்தத் தொடர்புகள் பின்வருமாறு கணக்கிடப்படுகின்றன:

### பாவ விலக்கு கணக்கீட்டுச் சார்பு (Kotlin Connection Function):
```kotlin
fun calculateKbConnection(x: Int, y: Int): String {
    if (x <= 0 || y <= 0) return "-"
    
    // முந்தைய வீடுகளைக் கணக்கிடுதல் (1-க்கு முந்தைய வீடு 12 ஆகும்)
    var prevY = y - 1
    if (prevY == 0) prevY = 12
    var prevX = x - 1
    if (prevX == 0) prevX = 12
    
    return when {
        x == prevY -> "$x"       // X வீடு Y வீட்டைத் துண்டிக்கிறது (உதாரணம்: Venus 10 & Rahu 11 -> 10)
        y == prevX -> "$y"       // Y வீடு X வீட்டைத் துண்டிக்கிறது
        x == y -> "$x"           // இரண்டுமே ஒரே வீடாக இருந்தால்
        else -> "$y, $x"         // விலக்கு விதி இல்லாத போது இரண்டும் இணைகிறது (உதாரணம்: Sun 11 & Saturn 8 -> 8, 11)
    }
}
```

### 1. கிரக அட்டவணை ரெண்டரிங் (Planet Table Rendering):
ஆப்பில் இந்தச் சார்பு ஒவ்வொரு கிரகத்திற்கும் `bhavaOccupied` ($x$) மற்றும் `starLordBhava` ($y$) ஆகியவற்றை `calculateKbConnection` அனுப்பி தொடர்பைக் கணக்கிட்டுத் திரையில் காட்டுகிறது:

```kotlin
val x = p.optInt("bhavaOccupied", 0)
val y = p.optInt("starLordBhava", 0)
val connection = calculateKbConnection(x, y)
```

### 2. பாவ அட்டவணை ரெண்டரிங் (Bhava Cusp Table Rendering):
அதேபோல், 12 பாவ முனைகளுக்கும் (House Cusps) அதன் நட்சத்திர அதிபதி நின்ற வீடும் ($a$), நின்ற நட்சத்திர அதிபதி நின்ற வீடும் ($b$) கொண்டு தொடர்பு கணக்கிடப்படுகிறது:

