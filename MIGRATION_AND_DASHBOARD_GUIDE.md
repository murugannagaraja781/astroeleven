# SQL to MongoDB Migration & Super Admin Dashboard Guide (தமிழ் வடிவம்)

இந்த ஆவணத்தில், உங்களுடைய `u453854592_astro.sql` கோப்பில் உள்ள தரவுகளை (Data) எவ்வாறு MongoDB-க்கு மாற்றுவது மற்றும் அந்தத் தரவுகளின் அடிப்படையில் **Super Admin Dashboard**-ல் என்னென்ன சேவைகள் (Services) மற்றும் பிரிவுகள் (Modules) தேவைப்படும் என்பதைப் பற்றி விரிவாகக் காண்போம்.

## பகுதி 1: SQL தரவை MongoDB-ல் ஏற்றுவது எப்படி? (SQL Data to MongoDB Migration)

SQL என்பது உறவுமுறை தரவுத்தளம் (Relational Database) ஆகும். MongoDB என்பது ஆவண-அடிப்படையிலான (Document-based NoSQL) தரவுத்தளம் ஆகும். எனவே, நேரடியாக SQL-ஐ MongoDB-ல் ஏற்ற முடியாது. அதை மாற்றுவதற்கு இரண்டு முக்கிய வழிகள் உள்ளன:

### வழிமுறை 1: Node.js மைக்ரேஷன் ஸ்கிரிப்ட் மூலமாக (பரிந்துரைக்கப்படும் வழி)
நாம் ஏற்கனவே ஒரு முழுமையான மைக்ரேஷன் ஸ்கிரிப்டை `scripts/migrate_sql_to_mongo.js` என்ற கோப்பில் உருவாக்கியுள்ளோம். இந்த ஸ்கிரிப்ட் `u453854592_astro.sql` கோப்பை வாசித்து, பயனர்கள், ஜோதிடர்கள், வாலட் பேலன்ஸ், கால் பதிவுகள், பேமெண்ட்கள், வித்ட்ராவல்ஸ், விளம்பரப் பதாகைகள் மற்றும் ஃபீட்பேக் ஆகியவற்றைத் தானாகவே MongoDB மாடல்களுக்கு (Mongoose models) ஏற்ப மேப் செய்து MongoDB-ல் சேமிக்கும்.

#### படி 1: தேவையான Packages நிறுவுதல்
மைக்ரேஷன் செய்வதற்கு தேவையான பேக்கேஜ்களை பின்வரும் கட்டளை மூலம் நிறுவிக் கொள்ளவும்:
```bash
npm install mysql2 dotenv
```

#### படி 2: மைக்ரேஷன் ஸ்கிரிப்டை இயக்குதல்

**A. டெவலப்மெண்ட் / சோதனை முறையில் இயக்க (Dev/Test Database-க்கு):**
சோதனை செய்யக்கூடிய பொதுவான (Working) MongoDB Atlas யூரிக்கு நேரடியாகப் பதிவேற்ற:
```bash
node scripts/migrate_sql_to_mongo.js --dev
```

**B. உங்களுடைய ப்ரொடக்ஷன் (Production Database-க்கு):**
உங்களுடைய `.env` கோப்பில் உள்ள `MONGODB_URI`-க்கு பதிவேற்ற:
```bash
node scripts/migrate_sql_to_mongo.js
```

> [!WARNING]
> **முக்கிய குறிப்பு (Production Run):**
> ப்ரொடக்ஷன் டேட்டாபேஸில் இயக்குவதற்கு முன், உங்களுடைய கணினியின்/சர்வரின் **IP முகவரியை (IP Address)** MongoDB Atlas கணக்கின் IP Whitelist-ல் சேர்த்துள்ளீர்களா என்பதை உறுதிப்படுத்திக் கொள்ளவும். இல்லையெனில் இணைப்பு பிழை (Server Selection Error) ஏற்படும்.

### வழிமுறை 2: CSV/JSON வழியாக இறக்குமதி செய்தல்
ஸ்கிரிப்ட் இல்லாமல் செய்ய விரும்பினால்:
1. உங்களுடைய SQL கோப்பை DBeaver அல்லது phpMyAdmin மூலம் **CSV** அல்லது **JSON** கோப்பாக மாற்ற வேண்டும்.
2. பின்னர் MongoDB Compass அல்லது `mongoimport` கட்டளையைப் பயன்படுத்தி இறக்குமதி செய்யலாம்:
```bash
mongoimport --db=astroeleven --collection=users --file=users.json --jsonArray
```

> [!IMPORTANT]
> **NoSQL சிறந்த நடைமுறை (Best Practice):** SQL-ல் உள்ள அனைத்து அட்டவணைகளையும் (Tables) தனித்தனி கலெக்ஷன்களாக மாற்றத் தேவையில்லை. உதாரணமாக, `users` மற்றும் `user_wallets` ஆகியவற்றை இணைத்து ஒரே பயனர் ஆவணத்தில் (User Document) நெஸ்டட் ஆப்ஜெக்ட்டாக (Nested Object) மாற்றுவது MongoDB-யின் செயல்திறனை அதிகரிக்கும்.

## பகுதி 2: Super Admin Dashboard-ல் தேவைப்படும் சேவைகள் (Dashboard Services Details)

உங்களுடைய SQL கோப்பில் உள்ள அட்டவணைகளை (Tables) ஆய்வு செய்ததில், ஒரு முழுமையான ஜோதிட செயலியை (Astrology App) நிர்வகிக்க **சூப்பர் அட்மின் டேஷ்போர்டில்** மொத்தம் **11 முக்கிய சேவைகள்/பிரிவுகள்** தேவைப்படுகின்றன:

### 1. பயனர்கள் மேலாண்மை (User Management)
செயலியைப் பயன்படுத்தும் வாடிக்கையாளர்களை நிர்வகிக்கும் பகுதி.
* **பின்னணி அட்டவணைகள் (SQL Tables):** `users`, `user_device_details`, `user_roles`, `roles`
* **டேஷ்போர்டில் இருக்க வேண்டியவை:**
  * பதிவுசெய்யப்பட்ட பயனர்களின் பட்டியல் (பெயர், மின்னஞ்சல், தொலைபேசி எண், மாநிலம்/நாடு).
  * பயனர்களைத் தற்காலிகமாக முடக்குதல் (Block/Unblock) மற்றும் நீக்குதல்.
  * பயனர்களின் சாதன விவரங்களை (Device Type, Token for Push Notification) பார்த்தல்.
  * வாடிக்கையாளர்களின் வாலட் பேலன்ஸ் (Wallet Balance) விவரங்களைச் சரிபார்த்தல்.

### 2. ஜோதிடர்கள் மேலாண்மை (Astrologer Management)
ஜோதிடர்களைப் பதிவு செய்து, அவர்களைக் கண்காணிக்கும் முதன்மைப் பகுதி.
* **பின்னணி அட்டவணைகள்:** `astrologers`, `astrologer_availabilities`, `astrologer_categories`, `astrologer_expertises`, `languages`, `degree_or_diplomas`
* **டேஷ்போர்டில் இருக்க வேண்டியவை:**
  * **புதிய ஜோதிடர்கள் ஒப்புதல் (Pending Approvals):** புதிய ஜோதிடர்கள் சமர்ப்பித்த சான்றிதழ்கள், ஆதார், பான் கார்டு ஆகியவற்றைச் சரிபார்த்து Approve அல்லது Reject செய்தல்.
  * ஜோதிடர்களின் சுயவிவர விவரங்கள் (Profile, Photo, Category, Expertise, Experience, Rate Per Minute).
  * ஜோதிடர்கள் பேசும் மொழிகள் மற்றும் அவர்களின் வாராந்திர வேலை நேரத்தை (Availabilities) மாற்றுதல்.
  * ஜோதிடரின் நிலையை (Online/Offline/Busy) நேரலையாகக் கண்காணித்தல்.

### 3. அழைப்பு பதிவுகள் (Call Logs & Tracking)
பயனர்களுக்கும் ஜோதிடர்களுக்கும் இடையே நடக்கும் ஆடியோ அழைப்புகளை நிர்வகித்தல்.
* **பின்னணி அட்டவணைகள்:** `callrequest`, `callaudio`, `user_call_histories`
* **டேஷ்போர்டில் இருக்க வேண்டியவை:**
  * அனைத்து அழைப்புகளின் விவரங்கள் (யார் அழைத்தது, எந்த ஜோதிடர், தேதி, நேரம், கால அளவு, கட்டணம்).
  * தவறிய அழைப்புகள் (Missed Calls) மற்றும் அதற்கான தானியங்கி ஆஃப்லைன் நிலையை கண்காணித்தல்.
  * ஆடியோ ரெக்கார்டிங் கோப்புகளைக் (Audio call files) கேட்டு ஆய்வு செய்யும் வசதி.

### 4. அரட்டை பதிவுகள் (Chat Logs & Tracking)
பயனர்களுக்கும் ஜோதிடர்களுக்கும் இடையே நடக்கும் அரட்டைகளைக் கண்காணித்தல்.
* **பின்னணி அட்டவணைகள்:** `chats`, `chatrequest`, `user_chats`, `user_chat_histories`, `assistantchat`, `livechat`
* **டேஷ்போர்டில் இருக்க வேண்டியவை:**
  * நேரலை அரட்டைகள் மற்றும் பழைய அரட்டை வரலாறுகளை (Chat History) வாசித்தல்.
  * அரட்டைக்கான கட்டணக் கழிவுகள் மற்றும் ஜோதிடரின் வருவாய் விவரங்கள்.
  * உதவி ஜோதிடர் (Assistant Chat) விவரங்களைக் கண்காணித்தல்.

### 5. அஸ்ட்ரோமால் மேலாண்மை (Astromall / E-commerce Management)
பூஜை பொருட்கள், ரத்தினக் கற்கள் போன்ற பொருட்களை விற்கும் பகுதி.
* **பின்னணி அட்டவணைகள்:** `astromall_products`, `product_categories`, `product_details`, `user_orders`, `order_addresses`, `order_payments`
* **டேஷ்போர்டில் இருக்க வேண்டியவை:**
  * தயாரிப்பு வகைகளை (Product Categories) உருவாக்குதல் மற்றும் தயாரிப்புகளைச் சேர்த்தல்/திருத்துதல்.
  * வாடிக்கையாளர்களின் ஆர்டர்களைக் (Orders) கண்காணித்தல் (Pending, Shipped, Delivered).
  * டெலிவரி முகவரிகள் மற்றும் ஆர்டர் கட்டண நிலைகளைச் சரிபார்த்தல்.

### 6. பணம் மற்றும் வாலட் பரிவர்த்தனைகள் (Payments & Wallet Transactions)
பயனர்களின் ரீசார்ஜ் மற்றும் நிதி பரிவர்த்தனைகளை நிர்வகித்தல்.
* **பின்னணி அட்டவணைகள்:** `payment`, `wallettransaction`, `user_wallets`, `rechargeamount`
* **டேஷ்போர்டில் இருக்க வேண்டியவை:**
  * பயனர்கள் வாலட்டில் செய்த ரீசார்ஜ் வெற்றியா அல்லது தோல்வியா என்பதைப் பார்த்தல்.
  * ரீசார்ஜ் பேக்கேஜ்களின் கட்டணங்களை (Recharge Offers / Amounts) நிர்ணயித்தல்.
  * ஒவ்வொரு வாலட் பரிவர்த்தனையின் முழுமையான லெட்ஜர் (Ledger) மற்றும் கேஷ்பேக் விவரங்கள்.

### 7. கமிஷன் மற்றும் தள்ளுபடி கூப்பன்கள் (Commissions & Coupons)
ஜோதிடர்களுக்கான கமிஷன் மற்றும் பயனர்களுக்கான விளம்பரக் குறியீடுகளை அமைத்தல்.
* **பின்னணி அட்டவணைகள்:** `commissions`, `commission_types`, `admin_get_commissions`, `coupons`
* **டேஷ்போர்டில் இருக்க வேண்டியவை:**
  * ஒவ்வொரு ஜோதிடருக்கும் நிர்வாகம் (Admin) பெற வேண்டிய கமிஷன் சதவிகிதத்தை நிர்ணயித்தல்.
  * புதிய விளம்பரக் குறியீடுகளை (Promo Coupons) உருவாக்குதல் (செல்லுபடியாகும் காலம், குறைந்தபட்ச/அதிகபட்ச ரீசார்ஜ் வரம்பு).

### 8. ஜோதிடர்களின் வருவாய் திரும்பப்பெறல் (Withdrawal Requests)
ஜோதிடர்கள் சம்பாதித்த பணத்தை வங்கியின் மூலம் திரும்பப் பெறக் கோரும் பகுதி.
* **பின்னணி அட்டவணைகள்:** `withdrawrequest`, `withdraws`, `withdrawmethods`
* **டேஷ்போர்டில் இருக்க வேண்டியவை:**
  * ஜோதிடர்களிடமிருந்து வரும் பணப் பரிமாற்றக் கோரிக்கைகளின் பட்டியல்.
  * அவர்களின் வங்கி விவரங்கள், UPI ஐடி மற்றும் பான் கார்டுகளைச் சரிபார்த்தல்.
  * பரிவர்த்தனையை Approve செய்து பணத்தை அனுப்பிய பின் "Completed" என மாற்றுதல்.

### 9. ராசி பலன் மற்றும் தினசரி கணிப்புகள் (Horoscope & Predictions)
பயனர்களுக்கான தினசரி ராசி பலன்களைப் பதிவேற்றுதல்.
* **பின்னணி அட்டவணைகள்:** `dailyhoroscope`, `dailyhoroscopeinsight`, `dailyhoroscopestatics`, `hororscope_signs`, `horoscopes`, `horoscope`
* **டேஷ்போர்டில் இருக்க வேண்டியவை:**
  * 12 ராசிகளுக்கான தினசரி, வாராந்திர, மாதாந்திர ராசி பலன்களை எழுதுதல் / திருத்துதல்.
  * ஆரோக்கியம், காதல், தொழில் போன்ற பிரிவுகளில் கணிப்பு விவரங்களைப் புதுப்பித்தல்.

### 10. வாடிக்கையாளர் ஆதரவு மற்றும் புகார்கள் (Customer Support & Tickets)
வாடிக்கையாளர்களின் பிரச்சினைகளைத் தீர்க்கும் பகுதி.
* **பின்னணி அட்டவணைகள்:** `tickets`, `help_supports`, `help_support_quations`, `help_support_quation_answers`, `ticketreview`
* **டேஷ்போர்டில் இருக்க வேண்டியவை:**
  * வாடிக்கையாளர்கள் எழுப்பும் புகார்கள் (Tickets) மற்றும் அதன் தற்போதைய நிலை (Open, Pending, Resolved).
  * அட்மின் புகார்களுக்குப் பதிலளிக்கும் வசதி.
  * அடிக்கடி கேட்கப்படும் கேள்விகளை (FAQ - Help Support Questions) நிர்வகித்தல்.

### 11. விளம்பரப் பதாகைகள் மற்றும் வலைப்பதிவு (Banners & Blogs Management)
ஆப்-ன் முகப்புப் பக்க விளம்பரங்கள் மற்றும் கட்டுரைகள்.
* **பின்னணி அட்டவணைகள்:** `banners`, `banner_types`, `blogs`, `blog_categories`, `blog_images`, `astrology_videos`
* **டேஷ்போர்டில் இருக்க வேண்டியவை:**
  * மொபைல் செயலியின் முகப்புப் பக்கத்தில் காட்டப்படும் பதாகைகளை (Promotional Banners) மாற்றுதல்.
  * ஜோதிடம் சார்ந்த கட்டுரைகளை (Blogs) படங்கள் மற்றும் வகைகளுடன் பதிவேற்றுதல்.
  * யூடியூப் வீடியோ இணைப்புகளைச் (Astrology Videos) செயலிக்குள் பதிவேற்றுதல்.
